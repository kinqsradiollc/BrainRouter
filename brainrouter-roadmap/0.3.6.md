## Items 1–8

**1. Goal-leakage fix (PR #26).** `readGoal` fell back to a workspace-scoped
`goal.json` on every new session, returning stale goals from previous sessions.
Fixed by making the session-scoped path authoritative; legacy path is now only
read on upgrade migration and then renamed out of reach.

**2. CLI shell redesign (PR #27).** Replaced three pre-banner `console.log` lines
with a boxed startup banner (workspace + hash, MCP, workflow, goal, session,
model). Added `/where` single-screen state view, persistent statusline with
`workflow`/`goal`/`plan`/`pr`/`exec`/`effort` segments, `--quiet`/`/quiet` toggle,
theme module (`dark`/`light`/`mono`), idle help hint, and a CJS bin shim to
suppress Node's `ExperimentalWarning` before ESM imports fire.

**3. Multi-workflow concurrency (PR #38).** Added `/workflow switch <slug>` to
refocus on an existing workflow, `/workflows` list with per-workflow status,
and last-used workflow hint in the banner on fresh sessions. Goals are now
strictly session-scoped (not per-workflow) — the per-workflow goal binding
design was tried and reverted because it reproduced the cross-session leak
from item 1.

**4. `ask_user_choice` (PR #30).** New local tool with an arrow-key picker
(↑/↓ navigate, ENTER confirm, SPACE toggle in multi-select), a synthetic
"Other → free-text" row, and eager input validation (2–4 options, no
duplicate labels). Non-TTY and silent-child contexts throw structured errors
instead of silently defaulting. Also fixed a latent `setRawMode(false)` bug
from the first CLI commit that was breaking Backspace and shift+tab.

**5. Reasoning-step capture (PR #32).** System-prompt rule tells the model to
call `memory_working_offload` with `kind: "reasoning"` after non-trivial tool
batches. Canvas renders reasoning nodes with a dashed border. Briefing surfaces
a "Recent reasoning (why-trail)" block capped at 3 most-recent entries to
avoid stuffing future turns. Also fixed: `ora`'s `discardStdin: true` was
dropping the readline keypress listener — routed all spinner calls through a
factory that forces `discardStdin: false` (PR #34, bundled).

**6. `/grill-me` (PR #31).** User-invoked slash command that primes the model
into CLARIFY mode for one turn — ask 2–5 questions via `ask_user_choice`
before any file edits or shell calls. Skips automatically when the current
workflow already has a `spec.md` (use `--force` to override). Cleared after
the turn so the overlay doesn't carry into the next prompt. Also bundled the
`agent.test.ts` split: 2,482-line monolith → 9 focused files under `src/tests/`.

**7. `/mode` + `/review-policy` (PR #35).** Consolidated the fragmented
`/yolo`/`/approve`/access-mode surface into two session-level knobs:
`executionMode: 'planning' | 'fast'` and `reviewPolicy: 'request' | 'proceed'`.
`/yolo on/off` becomes an alias that flips both. New `isDangerousCommand`
helper and pure `resolveRunCommandApproval` resolver (turned out neither
existed). Incidental safety fix: silent children can no longer run dangerous
commands regardless of `/yolo` state.

**8. `/effort` (PR #36).** Persisted `effort: 'low' | 'medium' | 'high'`
preference with `BRAINROUTER_EFFORT` env override. `medium` emits no overlay
(upgrade path is silent). Provider-aware `reasoning_effort` forwarding on Chat
Completions — detection keys on model name (not endpoint), covering OpenAI,
DeepSeek, OpenRouter, LM Studio 0.3.29+, and Ollama. Bundled: PR #37 removed
`resolveDefaultApiKey` (was peeking into MCP's private SQLite) and
`createDefaultConfig` (was fabricating a broken default on first run).

## Late-cycle additions (Items 9–11)

Added after a 2026-05-24 context-cost audit found the system prompt (~4,750 tokens)
and unconditional per-turn recall dominated per-turn spend.

**9. Context budget.** Trimmed `buildSystemPrompt()` from ~4,750 to ~1,400 tokens
by cutting static lecturing and moving tool mechanics into tool descriptions.
Gated recall behind first-turn / post-compaction / entity-keyword triggers
(`BRAINROUTER_RECALL_MODE=always|gated|off`). Reordered prompt so the static
prefix never changes turn-to-turn, maximising cache hits. Deduplicated goal
text: `goal-anchor` tagged system message is the single owner; continuation
prompt and briefing reference it rather than repeating it. Merged
`goal-budget-steering` into the anchor.

**10. MCP identity + offline UX.** Four-tier identity detection for the
BrainRouter MCP: explicit `identity` config field → name prefix → URL hostname
→ tool-signature fallback (`memory_recall` + `list_skills`). When BrainRouter
is offline the system prompt swaps the entire tools block for a one-line
offline notice, so the model stops attempting `memory_*` calls. Distinct
`brain` row in banner, statusline, and `/where` so "brain offline" is
distinguishable from "third-party MCP offline." Local-transcript fallback
(10d) deferred to 0.3.7.

**11. Multi-MCP foundation.** `McpClientPool` in `runtime/mcpPool.ts` holds a
map of `serverName → McpClientWrapper`. `listAllTools()` aggregates with
`mcp__<server>__<tool>` namespacing; collision policy prefers BrainRouter and
warns. New `/mcp list`, `/mcp reconnect [name]`, `/mcp tools [server]`
commands. Parallel cross-MCP tool execution deferred to 0.4.0.
