## Release Shape

- **Session correctness.** New sessions no longer inherit stale
  workspace-scoped goals. Goals are session-scoped; workflows are
  storage/navigation, not goal owners.
- **CLI usability.** Startup banner, persistent statusline, `/where`,
  themes, quiet mode, `/grill-me`, `ask_user_choice`, `/mode`,
  `/review-policy`, and `/effort` made the CLI less brittle and easier
  to steer.
- **Working memory.** Non-trivial reasoning can be offloaded through
  `memory_working_offload`, and briefings surface a capped recent
  "why-trail".
- **Context budget.** Static system prompt shrank from about 4,750 to
  about 1,400 tokens. Recall became gated, prompt ordering became more
  cache-stable, and repeated goal text collapsed into a single anchor.
- **MCP UX.** BrainRouter MCP detection uses config, name, URL, stdio,
  and tool-signature fallbacks. Offline mode is explicit in prompt,
  banner, statusline, and `/where`.
- **Multi-MCP foundation.** `McpClientPool` can aggregate server tools
  behind namespaced tool names. Full parallel cross-MCP execution was
  left for later releases.

## Build Order

1. Goal-leakage fix first, because it unblocked workflow correctness.
2. Shell redesign next, so workflow and MCP state had a visible home.
3. Picker/preference primitives: `ask_user_choice`, `/grill-me`,
   `/mode`, `/review-policy`, `/effort`.
4. Multi-workflow support once session state was stable.
5. Late-cycle context-budget, MCP identity, and multi-MCP foundation.

## Deferred

- Local transcript fallback when BrainRouter MCP is offline.
- Full parallel execution across multiple MCP servers.
