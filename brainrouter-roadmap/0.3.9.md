# 0.3.9 Roadmap — Memory Briefing + Cache-First Loop

**Status.** Items 1-14 are merged into `release/0.3.9` (PR #57). A
late-cycle configuration thread also shipped in commit `233dbc8`.

**Theme.** Land the final pre-0.4.0 CLI memory and cost-quality work
without starting federation, cross-CLI inboxes, or typed cross-vendor
delegation.

## Release Threads

| Thread | Goal | Status |
|---|---|---|
| A. Memory quality | Brief at the right moments, from the right sources, with inspectable decisions. | Shipped |
| B. Cache-first loop | Keep the prompt prefix byte-stable, pin memory into that prefix, repair bad tool calls, shrink old tool results, and show cost/cache telemetry. | Shipped |
| C. Config cleanup | Move CLI behavior knobs into `~/.config/brainrouter/config.json`, externalise catalogs, enrich LM Studio model metadata, and remove redundant fallback paths. | Shipped |

## Why This Release Exists

0.3.8 made delegation reliable. 0.3.9 makes long sessions cheaper and
less fragile:

- Memory recall is no longer a blunt first-turn event; it is routed by
  turn intent and source availability.
- Memory cards are pinned into a byte-stable prefix so memory continuity
  and provider prefix caching reinforce each other.
- Common tool-call failures are repaired before they waste a turn.
- Large old tool results shrink after they have served their purpose,
  while raw transcripts stay intact.
- The CLI now shows cache-hit ratio, token savings, and USD cost instead
  of making users calculate them.

## Implementation Summary

### Items 1-7 — Memory Quality

| # | Item | Shipped result |
|---:|---|---|
| 1 | Adaptive briefing trigger engine | Recall fires for first turn, continuation, post-compaction, file/debug/history cues, child synthesis, and active-goal refreshes. |
| 2 | Source-aware briefing router | Briefing routes across recall, working context, task state, explanations, file history, and failed attempts. |

### Items 8-14 — Cache, Repair, Cost

| # | Item | Shipped result |
|---:|---|---|
| 8 | Cache-first context regions | `ImmutablePrefix`, `AppendOnlyLog`, and `VolatileScratch` keep session prefix bytes stable. |
| 9 | Prefix-pinned memory | First-turn memory cards pin into the immutable prefix; mid-session re-briefings append instead of rewriting it. |
| 10 | Cache-hit telemetry | OpenAI-compatible and DeepSeek cache fields normalise into cache-hit ratio and saved-token stats. |
| 11 | Tool-call repair pipeline | Flatten, scavenge, truncation, and storm passes recover common malformed tool-call patterns. |
| 12 | Turn-end shrink | Oversized historical tool results are compacted after the turn; full raw output stays in transcripts. |
| 13 | Tier self-escalation | `<<<NEEDS_HIGH>>>` can retry a turn on the next model tier, bounded once per turn. |
| 14 | Cost + cache panel | `/tokens` and status UI expose per-turn cost, session cost, cache hit, and cache savings. |

## Acceptance

- `/briefing` explains why memory fired or skipped and which sources
  contributed.
- First-turn memory cards pin into the prefix; ordinary turns preserve
  the prefix fingerprint.
- `/refresh-memory`, model changes, MCP inventory changes, and skill
  changes intentionally re-pin memory.
- Tool-call repair reports scavenged, truncated, flattened, or stormed
  calls in verbose/debug surfaces.
- `/tokens` shows cache hit %, cost, and cache savings.
- CLI behavior config comes from `config.json`, not project `.env` files.

## Ship Gate

- `npm run build`
- `npm test`
- `npm test --workspace brainrouter-cli`
- Manual CLI smoke:
  - first prompt shows a briefing when BrainRouter MCP is online;
  - "thanks" skips full briefing and `/briefing` explains why;
  - continuation and file-specific prompts trigger useful source routing;
  - turn 2 shows cache-hit data in `/tokens`;
  - repeated identical tool calls are storm-suppressed;
  - `<<<NEEDS_HIGH>>>` triggers one visible tier-up retry.

## Out of Scope

- Federation: active sessions, session inbox, shared live discovery,
  cross-CLI messaging, and handoff.
- Typed cross-vendor `delegate_*` tools and `route_task`.
- Durable brain-agent job queue.
- Durable source chunk tables, full memory tree bucket sealing, vault
  mirror, and dashboard source tree browser.
- OAuth/integration provider surfaces.
