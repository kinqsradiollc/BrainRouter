# 0.3.8 Roadmap — CLI Delegation Reliability

**Status.** Shipped — 2026-05-26. Full release notes:
[`../brainrouter-changelog/0.3.8.md`](../brainrouter-changelog/0.3.8.md).

**Theme.** Make CLI child-agent delegation dependable before the larger
0.4.0 typed-delegation and federation work.

## What Shipped

| # | Item | Outcome |
|---:|---|---|
| 1 | Runtime child-drain guardrail | Parent turns no longer finish with prose-only "I am waiting" while child agents are still pending or running. |
| 2 | `task_agent` / `delegate_agent` split | Foreground task execution and background delegation now have separate tool names and clearer prompt policy. |
| 3 | Child progress in Ink | Child tool start/end rows and running-child status make long delegated work visible. |
| 4 | Parallel safe read-only tools | Independent reads run concurrently; writes, shell actions, and foreground tasks stay serialized. |
| 5 | Cron-style `/schedule` | Persistent cron and one-shot schedules with list/remove/enable/disable commands. |
| 6 | `/release-notes` | Bundled per-version notes render inside the CLI; `list` shows available versions. |
| 7 | Hooks JSON docs | `brainrouter-docs/hooks.md` documents hooks and hookify JSON with examples. |
| 8 | Strict tool-call recovery | Duplicate, malformed, unknown, and orphaned tool calls recover into well-formed tool-result flows. |
| 9 | Per-vendor MCP snippets | `/mcp install <vendor>` prints paste-ready config for supported clients. |
| 10 | Native Anthropic adapter | Direct `/v1/messages` support shipped in 0.3.8, then was removed in 0.3.9. |
| 12 | Prefix-aware memory tool lookup | Briefing and `/mcp` doctor detect prefixed BrainRouter tools correctly. |
| 13 | Ink question overlays | `ask_user_choice` and yes/no approvals render inside the Ink chat overlay. |

## Key Acceptance

- Spawning child agents cannot end in a fake waiting message; the parent
  drains children or reports their ids and statuses.
- `task_agent` blocks for foreground work; `delegate_agent` starts
  background work and returns a child id.
- Child progress is visible in the Ink scrollback and status row.
- Read-only tools can run in parallel without breaking tool-result order.
- `/schedule`, `/release-notes`, hook docs, MCP install snippets, and
  Ink question overlays are usable from the CLI.
- Briefing finds `mcp_<server>_<tool>` names and the single-underscore
  MCP tool form is canonical.

## Out of Scope

- Typed `delegate_*` tools, `route_task`, parent context snapshots, and
  output contracts: 0.4.0.
- Ownership contracts, tool budgeting, supervisor gates, packs, worker
  threads, and transcript debugger: 0.4.x.
- Brain-agent job queue, source chunks, memory tree, vault mirror, and
  proactive situation reports: MCP brain roadmap.
