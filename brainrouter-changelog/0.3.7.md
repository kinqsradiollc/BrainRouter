# 0.3.7 — 2026-05-26

Terminal UI redesign, in-terminal configuration, full Ink chat REPL, and
multi-agent registry foundations.

## Added

- **Full Ink chat REPL.** Banner, composer, scrollback, tool events,
  slash palette, footer, `/config`, `/login`, and `/init` now render
  through one Ink tree.
- **First-run wizard.** `Welcome → Theme → Provider → API key → Model →
  MCP → AGENT.md → Done` auto-runs when no config exists and is
  re-enterable via `/init`.
- **Settings and MCP manager.** Bare `/config` opens an arrow-key
  settings home; the MCP row supports list, add, edit, probe, and remove
  without restarting.
- **`/login`.** In-REPL MCP profile editor for transport, URL, key,
  reachability probe, save, and hot-connect.
- **`/model`.** Live model picker backed by the active endpoint's
  `/v1/models` response.
- **Multi-MCP pool.** The CLI connects to all configured servers on
  boot, merges tools into one inventory, and isolates server failures.
- **Data-driven agent registry.** Built-in agents moved to JSON files
  with built-in, user-global, and workspace override layers.
- **Spawn tier and depth rules.** Agent definitions carry a tier;
  worker agents cannot delegate and spawn depth is capped.

## Changed

- CLI credentials come from `~/.config/brainrouter/config.json`.
- MCP child stderr is piped so server logs do not bleed into the Ink UI.
- Identity-based MCP prefixes let BrainRouter tools survive profile
  renames.

## Fixed

- `/config provider <id>` now asks for that provider's API key.
- `/login` can update LLM credentials after initial MCP setup.
- HTTP MCP setup collects the BrainRouter API key for local and remote
  HTTP transports.
- Wizard Skip clears stale `activeServer` instead of reconnecting to an
  old profile.
