# 0.3.9 - Unreleased

Memory briefing quality **plus** Reasonix-inspired cache-first loop,
tool-call repair, and cost-control. The pre-0.4.0 release.

## Removed in 0.3.9

- **Anthropic native provider support.** The `/v1/messages` adapter
  added in 0.3.8-I6 (`brainrouter-cli/src/runtime/anthropicAdapter.ts`)
  and its supporting surfaces — provider catalog entry, `sk-ant-`
  known prefix, `ANTHROPIC_API_KEY` env-detect row, Anthropic
  cache-stat branch, Anthropic tier ladder, `claude-*` pricing rows,
  Anthropic-native runtime tests — were all removed. Claude models
  remain reachable via OpenAI-compatible gateways (OpenRouter,
  Together, Fireworks): set `provider: 'openai'` and point
  `endpoint` at the gateway base URL.
- The dedicated `brainrouter-cli/src/tests/anthropic-adapter.test.ts`
  file was deleted alongside the adapter.

## Reasonix-inspired (items 8–14, shipped)

- **Cache-first context regions.** Introduce
  `ImmutablePrefix` / `AppendOnlyLog` / `VolatileScratch` so the prefix
  is byte-stable across turns. SHA-256 prefix fingerprint exposed to
  tracing; cache-bust events log a `prefixChurn` warning with which
  field shifted. Re-pinned only on `/refresh`, model change, MCP tool
  inventory change, or active-skill latch change. (Cites
  `openSrc/DeepSeek-Reasonix/src/memory/runtime.ts`.)
- **Prefix-pinned memory briefing.** Move the briefing block out of
  the system message and into a pinned synthetic message pair at the
  prefix tail. The cards survive the prefix hash (anchor IDs are part
  of the fingerprint payload) so mid-session full re-briefings append
  to the log instead of busting cache. New `/refresh-memory` slash
  command; `BRAINROUTER_PREFIX_MEMORY_ANCHORS=on|off` env var.
- **Cache-hit telemetry.** Read `prompt_tokens_details.cached_tokens`
  (OpenAI / Groq / Fireworks / OpenRouter) and `prompt_cache_hit_tokens`
  (DeepSeek) into one normalised
  `{ cachedTokens, missedTokens, cacheHitRatio }` shape. Per-turn
  cache-hit % surfaces in `/tokens`. The Anthropic-native shape
  (`cache_read_input_tokens`) was dropped alongside the adapter.
- **Tool-call repair pipeline.** New `brainrouter-cli/src/agent/repair/`
  directory with four passes adapted from
  `openSrc/DeepSeek-Reasonix/src/repair/`:
  - `flatten` — schemas with >10 leaves or depth >2 are presented to
    the model in dot-notation form; arguments re-nested at dispatch
    time;
  - `scavenge` — recover tool calls leaked into `reasoning_content`
    (or DSML markup in regular content);
  - `truncation` — rebalance unbalanced JSON in tool arguments;
  - `storm` — suppress identical `(name, args)` repeats inside a
    sliding window; mutating calls clear prior read-only entries.
  Existing `toolCallRecovery.ts` helpers (dedupe / orphan synthesis /
  stalled preamble / suggest-similar-tool) remain alongside.
- **Turn-end tool-result auto-shrink.** Walk the log after each turn
  and replace any `role: tool` content over `TURN_END_RESULT_CAP_TOKENS`
  (default 3000) with the TokenJuice-lite compaction. Full raw outputs
  remain in the transcript layer. Proactive 40% context-ratio threshold
  triggers the same shrink mid-iteration.
  `BRAINROUTER_TURN_END_RESULT_CAP_TOKENS` env override.
- **Model-tier self-escalation.** When the model emits
  `<<<NEEDS_HIGH>>>` (or `<<<NEEDS_HIGH: <reason>>>>`) on the first line,
  the runtime aborts and retries on the next-higher provider tier.
  Tier ladders for OpenAI and DeepSeek built in; OpenRouter / generic
  / Claude-via-gateway configurable in
  `~/.config/brainrouter/config.json` (Anthropic native ladder removed
  alongside the adapter).
  Auxiliary calls (compaction, repair retry, child-agent flash spawn)
  always pin to the lowest tier. New `/tier <flash|standard|pro>` slash
  command. Silent escalation is rejected — every escalation surfaces
  in a yellow Ink warning row.
- **Cost + cache panel.** `/tokens` now shows colored per-turn USD,
  per-session USD, cache hit %, and cache savings vs no-cache
  baseline. New `brainrouter usage --since 7d` aggregates across all
  sessions. Pricing table in `brainrouter-cli/src/runtime/pricing.ts`
  for the three slim-catalog providers; LM Studio / Ollama priced at
  $0 by definition.

## Added

- Adaptive CLI memory briefing trigger engine for first turn, post-compaction, continuation, memory/history, file path, debug/retry, recent tool failure, active-goal periodic refresh, and child-agent synthesis cues.
- Source-aware briefing router across recall, working context, task state, recall explanations, file history, and failed attempts when those MCP tools are available.
- TokenJuice-lite CLI tool-output compaction before model-visible tool-result insertion, with full raw output still preserved in transcripts.
- `/briefing` inspector with decision, reasons, planned/queried sources, skipped sources, source stats, record IDs, warnings, injected tokens, and compacted chars avoided.
- `/memories sources [limit]` read-only source manifest spike for local code/docs discovery without durable source tables or vault writes.
- Local briefing benchmark tests for trigger behavior, compaction savings, and end-to-end `buildMemoryBriefing` execution with stub MCP across all six roadmap scenarios (first-turn, continuation, file-specific, debug retry, post-compaction, child synthesis).
- `memoryPolicy` module: `assessRecallCards` (stale/off-workspace warnings) and `assessCapturePayload` (secret-block) — local CLI policy, not an enterprise settings layer.

## Changed

### Provider catalog slimmed to OpenAI + LM Studio + Ollama

The wizard and `/config provider` picker now lists three rows. The DeepSeek, OpenRouter, Claude-via-OpenRouter, and Gemini-OpenAI-compat entries are gone — they were near-duplicates of OpenAI with different default endpoints, and the picker bloat was worse than the savings.

- **OpenAI doubles as the OpenAI-compatible custom URL flow.** Picking it now prompts for a base URL pre-filled with `https://api.openai.com/v1` but freely replaceable with DeepSeek, OpenRouter, Together, Groq, Fireworks, self-hosted vLLM, or any other `/v1` endpoint that speaks the OpenAI chat-completions shape. The separate "Other endpoint" picker option is removed since this covers it.
- **LM Studio and Ollama** keep their fixed loopback endpoints (no prompt for those — editing a localhost URL is a config.json edit).
- **`/models` picker is unchanged** — it already runs `/v1/models` against whatever endpoint is configured and falls back to the per-provider curated short-list, with "Other model" for manual entry. Works for all three remaining entries.
- **`looksLikeForeignModel` simplified** — only fires when the active endpoint is `api.openai.com` itself. Custom OpenAI-compat endpoints are vendor-agnostic by design.

### Memory + capture (carried from earlier 0.3.9 work)

- Memory capture now redacts user and assistant turn text before sending it to the memory capture tool, and refuses the capture entirely when the payload still contains credential-shaped tokens (`sk-…`, `ghp_…`, `AKIA…`, PEM private key blocks, Slack `xox` tokens).
- Recall card previews now run through `redactText` so the structured-card path matches the opaque-dump path on secret handling.
- Briefing warnings now flag stale/superseded/needs-verification records and off-workspace path references in recalled content.
- `/tokens` now reports tool-output compaction savings alongside child-output offload savings.

## Notes

- 0.3.9 intentionally does not add federation, source chunk tables, active-session registries, cross-CLI inboxes, durable brain-agent queues, or vault mirrors. Those remain 0.4.0 / 0.4.x work.
