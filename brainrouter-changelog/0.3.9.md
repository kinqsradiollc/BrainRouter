# 0.3.9 — 2026-05-28

Memory briefing quality, cache-first context, tool-call repair,
cost-control, and CLI configuration cleanup. This is the final
pre-0.4.0 release.

## Breaking / Removed

- **CLI `.env` loading removed.** Behavior settings now live under
  `cli.*` in `~/.config/brainrouter/config.json`. Credential env vars
  such as `OPENAI_API_KEY`, `BRAINROUTER_API_KEY`, and
  `BRAINROUTER_HOME` still work.
- **Native Anthropic provider removed.** The 0.3.8 `/v1/messages`
  adapter and its tests/catalog/pricing/cache surfaces were deleted.
  Claude remains supported through OpenAI-compatible gateways such as
  OpenRouter, Together, and Fireworks.
- **CLI recall fallback wrapper removed.** The brain-side recall
  pipeline now owns fallback behavior end to end.

## Added

- **Adaptive memory briefing.** Recall now triggers for first turns,
  continuations, post-compaction, file/history/debug cues, child-agent
  synthesis, and active-goal refreshes.
- **Source-aware briefing router.** Briefings can combine recall,
  working context, task state, recall explanations, file history, and
  failed-attempt context when the matching MCP tools exist.
- **`/briefing` inspector.** Shows decision reasons, planned and queried
  sources, skipped sources, source stats, warnings, record ids, injected
  tokens, and compaction savings.
- **Read-only source manifest.** `/memories sources [limit]` scans local
  code/docs into an ephemeral manifest without adding durable source
  tables or vault writes.
- **Cache-first context regions.** `ImmutablePrefix`, `AppendOnlyLog`,
  and `VolatileScratch` keep prefix bytes stable across turns and emit a
  fingerprint for tracing.
- **Prefix-pinned memory cards.** First-turn memory cards pin into the
  immutable prefix; mid-session re-briefings append refresh cards instead
  of rewriting the prefix. `/refresh-memory` repins intentionally.
- **Cache-hit telemetry.** OpenAI-compatible and DeepSeek cache fields
  normalise into cached tokens, missed tokens, cache-hit ratio, and saved
  token counts.
- **Tool-call repair pipeline.** New repair passes flatten deep schemas,
  scavenge calls leaked into content, repair truncated JSON, and suppress
  repeated identical calls.
- **Turn-end tool-result shrink.** Oversized historical `role: tool`
  entries compact after the turn while full raw outputs remain in
  transcripts.
- **Model-tier self-escalation.** `<<<NEEDS_HIGH>>>` can retry once on
  the next configured tier; `/tier flash|standard|pro|auto` controls the
  session pin.
- **Cost + cache panel.** `/tokens` shows per-turn cost, session cost,
  cache hit %, and cache savings. Pricing can be overridden at
  `~/.config/brainrouter/pricing.json`.

## Changed

- **CLI config consolidated.** Former `BRAINROUTER_*` behavior knobs
  moved into typed `cli.*` fields in `config.json`, loaded through
  `getCliKnobs()` with test/argv overrides via `setCliKnobOverride()`.
- **Model/provider catalogs externalised.** `brainrouter-cli/config/`
  now contains `models.json`, `providers.json`, and
  `api-key-prefixes.json`, so catalog updates no longer require code
  edits.
- **LM Studio model metadata enriched.** When using
  `http://localhost:1234/v1`, `/status` and the wizard show native
  `/api/v1/models` details such as loaded state, params, quantisation,
  format, tool-use training flag, and reasoning options.
- **Token estimates improved.** Fallback token counting now accounts for
  CJK, code-heavy, and prose-heavy text.
- **Provider picker slimmed.** `/config provider` now presents OpenAI,
  LM Studio, and Ollama. OpenAI doubles as the custom OpenAI-compatible
  base URL flow for DeepSeek, OpenRouter, Groq, Fireworks, Together,
  vLLM, and similar endpoints.
- **Memory capture hardened.** Capture redacts obvious secrets and
  refuses payloads that still contain credential-shaped tokens. Recall
  previews use the same redaction path.
- **Memory policy warnings added.** Briefing warns on stale,
  superseded, needs-verification, and off-workspace recalled records.

## Tests / Verification

- Briefing benchmarks cover first-turn, continuation, file-specific,
  debug retry, post-compaction, and child-synthesis scenarios.
- Cache stats, repair passes, config loading, pricing, and tier
  escalation have targeted CLI tests.

## Notes

0.3.9 intentionally does not add federation, source chunk tables,
active-session registries, cross-CLI inboxes, durable brain-agent
queues, or vault mirrors. Those remain 0.4.0 / 0.4.x work.
