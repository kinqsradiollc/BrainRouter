# 0.3.9 - Unreleased

Memory briefing quality release before the 0.4.0 federation work.

## Added

- Adaptive CLI memory briefing trigger engine for first turn, post-compaction, continuation, memory/history, file path, debug/retry, recent tool failure, active-goal periodic refresh, and child-agent synthesis cues.
- Source-aware briefing router across recall, working context, task state, recall explanations, file history, and failed attempts when those MCP tools are available.
- TokenJuice-lite CLI tool-output compaction before model-visible tool-result insertion, with full raw output still preserved in transcripts.
- `/briefing` inspector with decision, reasons, planned/queried sources, skipped sources, source stats, record IDs, warnings, injected tokens, and compacted chars avoided.
- `/memories sources [limit]` read-only source manifest spike for local code/docs discovery without durable source tables or vault writes.
- Local briefing benchmark tests for trigger behavior and compaction savings.

## Changed

- Memory capture now redacts user and assistant turn text before sending it to the memory capture tool.
- `/tokens` now reports tool-output compaction savings alongside child-output offload savings.

## Notes

- 0.3.9 intentionally does not add federation, source chunk tables, active-session registries, cross-CLI inboxes, durable brain-agent queues, or vault mirrors. Those remain 0.4.0 / 0.4.x work.
