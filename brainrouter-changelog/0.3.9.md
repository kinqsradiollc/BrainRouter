# 0.3.9 - Unreleased

Memory briefing quality release before the 0.4.0 federation work.

## Added

- Adaptive CLI memory briefing trigger engine for first turn, post-compaction, continuation, memory/history, file path, debug/retry, recent tool failure, active-goal periodic refresh, and child-agent synthesis cues.
- Source-aware briefing router across recall, working context, task state, recall explanations, file history, and failed attempts when those MCP tools are available.
- TokenJuice-lite CLI tool-output compaction before model-visible tool-result insertion, with full raw output still preserved in transcripts.
- `/briefing` inspector with decision, reasons, planned/queried sources, skipped sources, source stats, record IDs, warnings, injected tokens, and compacted chars avoided.
- `/memories sources [limit]` read-only source manifest spike for local code/docs discovery without durable source tables or vault writes.
- Local briefing benchmark tests for trigger behavior, compaction savings, and end-to-end `buildMemoryBriefing` execution with stub MCP across all six roadmap scenarios (first-turn, continuation, file-specific, debug retry, post-compaction, child synthesis).
- `memoryPolicy` module: `assessRecallCards` (stale/off-workspace warnings) and `assessCapturePayload` (secret-block) — local CLI policy, not an enterprise settings layer.

## Changed

### Provider catalog slimmed to OpenAI + LM Studio + Ollama

The wizard and `/config provider` picker now lists three rows. The DeepSeek, OpenRouter, Claude-via-OpenRouter, and Gemini-OpenAI-compat entries are gone — they were near-duplicates of OpenAI with different default endpoints, and the picker bloat was worse than the savings.

- **OpenAI doubles as the OpenAI-compatible custom URL flow.** Picking it now prompts for a base URL pre-filled with `https://api.openai.com/v1` but freely replaceable with DeepSeek, OpenRouter, Together, Groq, Fireworks, self-hosted vLLM, or any other `/v1` endpoint that speaks the OpenAI chat-completions shape. The separate "Other endpoint" picker option is removed since this covers it.
- **LM Studio and Ollama** keep their fixed loopback endpoints (no prompt for those — editing a localhost URL is a config.json edit).
- **`/models` picker is unchanged** — it already runs `/v1/models` against whatever endpoint is configured and falls back to the per-provider curated short-list, with "Other model" for manual entry. Works for all three remaining entries.
- **`looksLikeForeignModel` simplified** — only fires when the active endpoint is `api.openai.com` itself. Custom OpenAI-compat endpoints are vendor-agnostic by design.

### Memory + capture (carried from earlier 0.3.9 work)

- Memory capture now redacts user and assistant turn text before sending it to the memory capture tool, and refuses the capture entirely when the payload still contains credential-shaped tokens (`sk-…`, `ghp_…`, `AKIA…`, PEM private key blocks, Slack `xox` tokens).
- Recall card previews now run through `redactText` so the structured-card path matches the opaque-dump path on secret handling.
- Briefing warnings now flag stale/superseded/needs-verification records and off-workspace path references in recalled content.
- `/tokens` now reports tool-output compaction savings alongside child-output offload savings.

## Notes

- 0.3.9 intentionally does not add federation, source chunk tables, active-session registries, cross-CLI inboxes, durable brain-agent queues, or vault mirrors. Those remain 0.4.0 / 0.4.x work.
