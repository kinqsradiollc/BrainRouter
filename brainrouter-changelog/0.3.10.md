# 0.3.10 — Unreleased

Native Anthropic Messages API parity. The 0.3.8 native adapter shipped
the minimum (system hoist, tool_use/tool_result, max_tokens, opt-in
caching, extended thinking). 0.3.10 closes the remaining gaps so the CLI
fully supports the official Claude API surface short of SSE streaming.

## Added

### Request side

- **Sampling overrides.** `temperature`, `top_p`, `top_k`,
  `stop_sequences` (auto-sliced to Anthropic's 4-entry cap) forwarded
  per-call or via new optional `LLMConfig` fields.
- **`metadata.user_id`** forwarded for per-user attribution.
- **Multimodal user turns.** When `message.content` is a structured
  block array (e.g. `image`, `document`, `text`) the adapter passes it
  through verbatim instead of force-stringifying. Image and document
  prompts now reach the API correctly.
- **1h cache TTL.** `cache_control.ttl='1h'` is selectable via build
  option or `BRAINROUTER_ANTHROPIC_CACHE_TTL=1h`. The
  `extended-cache-ttl-2025-04-11` beta header is added automatically.
- **Cache breakpoint on tools.** `cacheTools` option (or
  `BRAINROUTER_ANTHROPIC_CACHE_TOOLS=1`) adds a breakpoint to the last
  tool definition so large stable tool lists also cache.
- **Pluggable beta headers.** `BRAINROUTER_ANTHROPIC_BETA` is a
  comma-separated list forwarded as `anthropic-beta`. Required for
  interleaved thinking, fine-grained tool streaming, PDFs, etc.

### Response side

- **`stop_reason` propagation.** Callers can now distinguish
  `end_turn`, `tool_use`, `max_tokens`, `stop_sequence`, `pause_turn`,
  `refusal`. Truncations and refusals no longer look identical to
  normal completions.
- **`rawAssistantBlocks`.** Verbatim assistant content array returned
  so callers doing multi-turn extended thinking with tool_use can echo
  back `thinking`-with-`signature` and `redacted_thinking` blocks. The
  API rejects the follow-up turn otherwise.
- **Cache usage in metering.** `cache_read_input_tokens` and
  `cache_creation_input_tokens` flow through `usage` when the API
  reports them.
- **`AnthropicApiError`.** Typed error with `status`, `errorType`, and
  `retryAfterMs` parsed from the `retry-after` header (numeric seconds
  and HTTP-date forms). Lets callers back off cleanly on 429
  (`rate_limit`) and 529 (`overloaded`).

### Config + discoverability

- **`/config` keys.** `temperature`, `top-p`, `top-k`, `max-tokens`,
  `stop-sequences`, `metadata-user-id`, `anthropic-cache`,
  `anthropic-cache-ttl`, `anthropic-cache-tools`, `anthropic-beta` are
  now scriptable as `/config <key> <value>`. Use `-` / `unset` to clear.
  Validation: temperature ∈ [0,2], top-p ∈ [0,1], top-k ∈ [1,500] int,
  max-tokens ∈ [1,200000] int, stop-sequences capped at 4 entries,
  cache-ttl ∈ {5m, 1h}.
- **Persisted Anthropic settings.** `llm.anthropic = { cache, cacheTtl,
  cacheTools, beta }` lives in `~/.config/brainrouter/config.json` so
  `/config` choices survive restarts. Env vars
  (`BRAINROUTER_ANTHROPIC_CACHE`, `_CACHE_TTL`, `_CACHE_TOOLS`,
  `_BETA`) still win for per-shell overrides.
- **Interactive home row.** `/config` (bare) now shows a "Claude API
  options" row summarising the active Anthropic settings; selecting it
  prints the full table + env override reminders.

## Notes

- **Streaming (`stream: true`) is still out of scope.** The agent loop
  expects a single response object. SSE streaming requires an
  agent-loop refactor and is tracked separately.
- **Built-in server tools** (web_search, code_execution, computer_use)
  are not wired here. The pluggable beta header makes them experiment-
  able from the env without further code changes.
- OpenAI-compat paths are unchanged. The native adapter still only
  activates on Anthropic profile + `api.anthropic.com` (or
  `BRAINROUTER_ANTHROPIC_NATIVE=1` override).
