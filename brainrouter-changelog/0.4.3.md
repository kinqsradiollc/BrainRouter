# 0.4.3 — In flight

Brain Phases 2-5 + carried CLI/infra work. Notes accrue here as items land on
`release/0.4.3`; publish happens at the 0.4.3 ship gate.

## Added — so far

- **`/rewind --files`** — file-restore mode. An append-only undo log captures
  each file's prior content the first time it's mutated in a turn (write_file
  / edit_file / apply_patch); `/rewind <n> --files` reverts files changed after
  turn n to their turn-n state — **previewed + confirmed** before any disk
  write — then forks the conversation. No git dependency.
- **`/context` context-window header** — leads with the current fill
  (`~used / max`, % used, remaining) + the auto-compact threshold, on top of
  the cumulative per-skill / per-tool / per-briefing breakdown. The fill reuses
  the exact signal auto-compact triggers on (`getCurrentContextTokens`).

## Fixed — post feature-complete hardening

A focused investigation (pricing · config · recall quality · source retention)
turned up four issues on top of feature-complete 0.4.3; each fix shipped as its
own PR with tests.

- **Cost telemetry stuck at `$0.00`.** `pricingFor()` did an exact model-id
  lookup and returned a zero-cost row on a miss — it never walked
  `familyFallbacks` the way its sibling `contextWindowFor()` does, so any
  unlisted SKU (e.g. `gpt-5.3-codex`) priced to $0. Wired the family-fallback
  walk in (and broadened the codex fallback), and made `ModelPricing.inputCacheHit`
  optional + coerced — a latent `NaN` (two rows omit it) that also rendered as
  `$0.00`. The optional `~/.config/brainrouter/pricing.json` override was a red
  herring; the shipped table now covers dot-versioned SKUs.
- **`/status` crash (#59).** `Cannot read properties of undefined (reading
  'type')` when `activeServer` was empty while server profiles existed
  (`servers[''] === undefined`). Added an in-memory config self-heal (pick a
  sane profile — `identity:'brainrouter'` → `brainrouter`-prefixed → first;
  reads never write to disk) + a guarded `describeActiveServer`. `/debug-config`
  now prints resolved knob *values* in place of a stale `BRAINROUTER_*` env list
  (those CLI env vars were retired in the 0.3.9 knob migration). Deliberately
  did **not** backfill `cli.*` defaults into the file — that collapses the
  `config > preference > default` layering for knobs like `effort`/`theme`.
- **Recall pollution.** A "find all the vulnerabilities in our api" query
  surfaced five near-identical "BrainRouter is an autonomous agent" records over
  the real `bug_finding` results. Fixed in two parts. *Scoring:* the
  security-intent classifier now uses plural/synonym vocab and is checked early
  (it had matched only singular "vulnerability" after debug/review, so the query
  fell through to `build` — which *boosts* the boilerplate); plus a per-type
  `recallPriorityCap` so never-decaying generic types (`instruction`,
  `architecture_decision`, `task_state`) can't out-rank fresh findings.
  *Selection:* on the default no-cross-encoder path, a local, no-network
  lexical-relevance demotion + MMR-diversity pass demotes off-topic boilerplate
  and collapses near-duplicates (`BRAINROUTER_RECALL_DIVERSITY`). **Zero added
  recall latency** — token-set math only, no model/network call.
- **Unbounded transcript sources.** Every `/sources` row is an auto-ingested
  per-turn transcript with no retention. Added `memory_prune_sources` — a
  provenance-safe prune that deletes old `transcript` documents but never one
  whose chunks still back a live memory (so `memory_verify`/provenance never
  breaks), plus an `(user_id, created_at)` index; and the `/sources` dashboard
  view now hides transcripts by default behind a toggle.
- **`.env.example` completeness.** Documented the previously-undocumented brain
  vars (background jobs / dedup / session+inbox sweepers) and the new recall
  selection knobs, so the example matches what the server actually reads.
