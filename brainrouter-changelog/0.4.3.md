## Added — so far

- **`/rewind --files`** — file-restore mode. An append-only undo log captures
  each file's prior content the first time it's mutated in a turn (write_file
  / edit_file / apply_patch); `/rewind <n> --files` reverts files changed after
  turn n to their turn-n state — **previewed + confirmed** before any disk
  write — then forks the conversation. No git dependency.
- **`/context` context-window header** — leads with the current fill
  (`~used / max`, % used, remaining) + the auto-compact threshold, on top of
  the cumulative per-skill / per-tool / per-briefing breakdown. The fill reuses
  the exact signal auto-compact triggers on (`getCurrentContextTokens`).
