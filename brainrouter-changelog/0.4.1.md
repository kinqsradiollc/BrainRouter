# 0.4.1 — Unreleased

Federation handoff, multi-agent ownership + budgeting, the deferred
A1–A5 augmentations, and the brain-side job queue. This file accrues
notes as each track lands on the 0.4.1 line.

## Added

- **Brain-side job queue + agent registry (BRAIN-P1, partial).** The
  eight pipeline stages that already run inside the memory engine
  (`cognitive_extractor`, `memory_deduper`, `contradiction_checker`,
  `graph_extractor`, `focus_shift_judge`, `focus_distiller`,
  `identity_distiller`, `relevance_judge`) are now formalised as a
  data-driven registry, and every brain-side action can be tracked as a
  durable, retryable row.
  - **`memory_jobs` table** (BRAIN-DESIGN-T2) — `pending → running →
    done|failed|cancelled` lifecycle with priority, attempts, and a
    `runAfter` "do not run before" signal. Cross-process-safe claim
    (`BEGIN IMMEDIATE` under WAL) so federated brain processes can't
    double-pick a job.
  - **Brain agent registry** (BRAIN-DESIGN-T1) —
    `listBrainAgents()` / `findBrainAgentById()` over the eight locked
    agent definitions (id, modelClass, reads/writes, dependsOn,
    idempotencyKey).
  - **Scheduler primitives** — exponential backoff (`2^attempts × 30s
    + jitter`, capped at 5 min) and an idempotency-deduped
    `enqueueAgentJob` that refuses a second job with the same key while
    one is pending/running.
  - **MCP tools** (BRAIN-DESIGN-T3) — `memory_agent_status` (read-only
    per-agent rollup: last status, 24h success rate, pending count),
    `memory_agent_run` (idempotent enqueue), `memory_job_retry`
    (re-arm a failed/cancelled job).

  Still to come on this track: wiring the live pipeline stages onto each
  agent's `execute()` + the runner loop (BRAIN-P1-T3), and the
  dashboard/CLI health surface (BRAIN-P1-T5).
