# 0.3.8 — 2026-05-26

CLI delegation reliability, parallel reads, and quick wins carried from
0.3.7.

## Added

- **Child-drain guardrail.** Parent turns no longer finish with
  prose-only "I am waiting" while spawned children are still pending or
  running. Timeouts return child ids, statuses, summaries, and a
  `/continue` hint.
- **`task_agent` / `delegate_agent`.** Foreground child work now blocks
  through `task_agent`; background work returns a running id through
  `delegate_agent`.
- **Child progress in Ink.** Child tool start/end rows show id, role,
  tool, duration, success/error, and a short summary.
- **Parallel safe reads.** Independent read-only tool calls run
  concurrently while writes, shell, and orchestration stay serialized.
- **Cron `/schedule`.** Persistent cron and one-shot schedules with
  list, remove, enable, and disable commands.
- **`/release-notes`.** Current and past release notes render inside the
  Ink REPL; `list` shows bundled versions.
- **Strict tool-call recovery.** Duplicate, malformed, unknown, and
  orphaned tool calls recover into well-formed tool-result envelopes.
- **Per-vendor MCP install snippets.** `/mcp install <vendor>` prints
  paste-ready config for supported MCP clients.
- **Native Anthropic adapter.** Direct `/v1/messages` support shipped in
  0.3.8. This was later removed in 0.3.9.
- **Hooks JSON docs.** `brainrouter-docs/hooks.md` documents hooks and
  hookify JSON with worked examples.
- **Ink question overlays.** `ask_user_choice` and yes/no confirmations
  render inside the active chat overlay.

## Fixed

- Briefing now detects prefixed BrainRouter memory tools such as
  `mcp_brainrouter_memory_recall`.
- `mcp_<server>_<tool>` is the canonical MCP tool-name form;
  double-underscore names are normalised at the pool boundary.
- Child-drain guardrail tracks `task_agent` and `delegate_agent`.
- Read-only access mode permits `task_agent` and `delegate_agent`.
- `delegate_agent` preserves errors and child ids from failed spawns.
- `handleWait` clears timeout handles on every resolution path.
- Garbled `BRAINROUTER_MAX_SPAWN_DEPTH` values fall back to the default.
- JWT verification compares base64url signatures in string space,
  catching padding-bit tampering.

## Deferred

- `@mentions` tab completion moved to 0.4.x.
