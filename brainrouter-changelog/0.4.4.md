# 0.4.4 — In progress

Memory pipeline tightening + unified exec policy. 0.4.3 shipped the depth
primitives (source chunks, provenance, blackboard, tree, vault, code chunker,
benchmark gate) and wired every brain agent; 0.4.4 makes them **precise and
unavoidable** rather than available-but-optional. Theme: *tighten, don't
invent.* (The CLI multi-agent + parity feature bar this sits on top of shipped
across 0.4.0–0.4.2.)

## Shipped — so far

- **Exact chunk-level provenance (MEM-15)** — capture now links each cognitive
  record only to the source chunk(s) it actually derives from, attributed by
  salient-token overlap (`memory/source/attribution.ts`), replacing the 0.4.3
  batch-level linking that tied every record in a window to every chunk.
  `memory_verify` / `memory_provenance` surface the precise chunks. Deterministic,
  zero-LLM; tunable via `BRAINROUTER_PROVENANCE_FLOOR` / `_MAX_CHUNKS`.
- **Blackboard-default admission (MEM-16)** — extracted candidates are staged on
  the blackboard and reconciled (batch dedup + below-threshold reject) before any
  commit; only survivors proceed to the existing upsert/embed/contradiction/graph
  path, each stamped committed for audit. Duplicate/rejected candidates stay on
  the blackboard (`memory_blackboard_review`), out of long-term memory. Gated by
  `BRAINROUTER_BLACKBOARD_ADMISSION` (default on); fail-open.

- **Recall expansion refs inline (MEM-17)** — recall hits now carry their
  precise `sourceChunkIds` (post MEM-15) and a covering `treeNodeId`, and the
  `<relevant-memories>` briefing shows a compact `↳ source: … · tree: …` hint
  per memory. The tools guide points at `memory_fetch_source_chunk` /
  `memory_tree_walk`, so a client can drill from a summary to its exact source
  in one hop instead of a second blind query.

- **Parser-backed code chunking (MEM-18)** — the regex first-cut (symbol starts
  only, no real end) is replaced by a char-level structural scan: a brace
  matcher for C-family/Rust (ignoring braces in strings/comments, ending at the
  closing brace or a brace-less item's `;`) and an indentation scan for Python.
  Top-level code between symbols becomes its own region. A `CodeStructureAdapter`
  seam lets a tree-sitter/LSP adapter slot in later; no new runtime deps.

## Planned

- **Benchmark modes + deterministic recall config (MEM-19)** — the retrieval
  benchmark now passes each mode's config to recall PER-CALL (`limitsOverride` /
  `selectionOverride` / `disableReranker` / `disableJudge`) instead of mutating
  `process.env` and restoring it — fixing a global-state leak that flaked the
  harness. Modes: `baseline`, `lexmmr`, plus `rerank` (when a reranker is
  configured) and `judge` (when the relevance judge is enabled); unavailable
  augmentations are reported as skipped, not silently counted as baseline. The
  strict recall-floor gate is retained. (Chunk/tree "AST" retrieval is a
  separate fixture, not scored by this record-self-retrieval harness.)

### Tree

- **Tree policy split** — separate generic tree mechanics from source/topic/
  global domain policy; document + schedule the three domains.

### Trust

- **Unified exec policy** — route file-write / edit / apply_patch / child-spawn
  / background / network through the one `decideExecutionPolicy` surface (only
  the shell tool does today), with approval audit events; non-TTY fails closed.

### Governance & retention

- **Governance dry-run expansion** to source chunks / tree nodes / vault
  exports / offload refs with per-class reclaim estimates.
- **Offload reclaimer** — retention that protects active refs, removes orphans,
  logs reclaim stats.
- **Redaction regression fixtures** across transcript / source-chunk /
  blackboard / vault / offload write paths.

### CLI

- **`/reload-skills`** — explicit skill-directory force re-scan.
- Optional carried follow-ups (capacity permitting): interactive `/agents
  create` wizard, in-flight `/bg` detach, post-edit LSP diagnostics.
