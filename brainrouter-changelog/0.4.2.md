# 0.4.2 — Cross-vendor federation + Multi-agent Phases 5-6 + durable workflows + full CLI parity (2026-05-30)

The CLI-and-federation release. Cross-vendor task delegation (Federation
Stage 5), CLI multi-agent Phases 5 and 6 (review fan-out, progressive result
handoff, agent-definition packs, dedicated worker threads, delegation-aware
memory capture, brain-agent provenance), a durable **workflow run engine**
with a live `/workflows` viewer, the full **CLI-parity batch**, version
centralization, a docs refresh with an MCP API reference, and two timeline /
context ergonomics commands.

> **Scope note.** This release ships the CLI / federation / multi-agent
> story. The heavier brain-side work-streams that were originally slated here
> — Agent transcript debugger (§6.5), Brain-Side Phase 2 (token-aware capture)
> and Phase 3 (source chunks + vault mirror) — move to **0.4.3** alongside
> Brain Phases 4-5, so the brain phases land together. Three 0.4.x infra items
> (git-worktree isolation, AST-aware chunking, the benchmark harness) also
> move to 0.4.3.

## Federation Stage 5 — cross-vendor delegation

- **`session_delegate_task` / `session_delegations`** MCP tools + the CLI
  **`/handoff`** command. Hand a task to another CLI/vendor on the same brain,
  addressed directly or by `<clientKind>:next-idle` (e.g. `codex:next-idle`)
  so the brain routes to the next idle peer of that kind. No live peer? The
  task is durably queued in the new `pending_delegations` table and delivered
  when one registers.

## CLI multi-agent — Phases 5 & 6

- **Confidence-scored review fan-out** (`/review-auto`) — parallel reviewers
  return findings with a confidence; the parent dedupes by
  `(file, line-range, root-cause)` and filters below a threshold.
- **Progressive result handoff** — large child outputs stream back through a
  structured result channel instead of being pasted whole into the parent.
- **Agent-definition packs** (`/pack`) — reusable role bundles (e.g.
  `pr-review`, `feature-dev`) enabled opt-in, preserving the base registry.
- **Dedicated worker threads** (`/workers`) — detached, file-backed background
  agents under `.brainrouter/cli/workers/<id>/`; dead-process workers are
  reconciled to `failed` on startup.
- **Delegation-aware memory capture** — five typed memory-event kinds so
  delegated work is attributable in recall.
- **Brain-agent awareness + provenance** — `memory_provenance` (the "why is
  this memory here?" trail) behind `/brain why`.

## Durable workflows + live viewer

- **Workflow run engine** — a persisted per-step `run.json` ledger per
  workflow folder, lazily created when the agent first reports progress via
  the new `workflow_progress` tool, and reconciled on restart (a `running`
  run from a dead process flips to `interrupted`).
- **`/workflows [slug]` live viewer** — per-workflow run status + a step
  glyph strip + done/total in the list; `<slug>` drills into the per-step
  timeline (glyph, title, duration, note).
- **Idle background-completion notifications** — when a child agent / worker /
  workflow run finishes while you're idle, a notice prints above the prompt
  (`cli.notifyBell` optionally rings the terminal bell).

## CLI parity batch

- **`/review --fix`** — applies the surviving high-signal fixes in place +
  runs build/tests, reverting any regression.
- **`/simplify [--dry-run]`** — a first-class behavior-preserving
  code-simplification pass.
- **`/effort xhigh`** (alias **`max`**) — an extra-high reasoning level above
  `high`; `/model <name> --session` switches the live model without
  persisting; **`cli.fallbackModel`** auto-swaps + retries once on a
  model-not-found error.
- **`!` shell escape** — `! <command>` runs a shell command straight from the
  composer (sandboxed when `cli.sandbox=on`; no approval gate — you typed it).
- **`/rewind [n]`** — lists the last 20 turns; `/rewind <n>` forks a new
  session truncated to that turn (branch the conversation from an earlier
  point).
- **`/context [all|current]`** — token breakdown: total (parent + children) +
  per-skill (bucketed by active skill) + per-briefing + per-tool call counts.

## Maintenance

- **Version is now read from each package's own `package.json`** through a
  single `version.ts` per package — bumping a release is one edit and can't
  drift. Fixed stale `0.3.8` hardcodes in the MCP server/client handshake and
  the CLI `--version` flag.
- **Docs refresh** — README + BRAINROUTER.md (now with an **MCP API
  reference**) + the `brainrouter-docs/` set brought current; removed
  references to features that were already retired (the native Anthropic
  `/v1/messages` adapter, `route_agent`, the `BRAINROUTER_EFFORT` env var).
